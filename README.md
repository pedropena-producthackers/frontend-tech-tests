# Frontend Tech Tests — La Razón

Pruebas técnicas para entrevistas de Frontend Developer. Cada carpeta es un sandbox independiente, abrible directamente en CodeSandbox.

## Cómo usar

El candidato abre la URL correspondiente, CodeSandbox crea un fork y trabaja en su propia copia.

| Prueba | Tema | Tiempo | Abrir en CodeSandbox |
|---|---|---|---|
| A | Tarjeta de artículo SEO-friendly (HTML/CSS) | 25 min | [Abrir](https://codesandbox.io/s/github/pedropena-producthackers/frontend-tech-tests/tree/main/prueba-a-article-card) |
| B | News Feed con consumo de API (JS vanilla) | 25 min | [Abrir](https://codesandbox.io/s/github/pedropena-producthackers/frontend-tech-tests/tree/main/prueba-b-news-feed) |
| C | Server Action + Formulario en Next.js | 25 min | [Abrir](https://codesandbox.io/p/sandbox/github/pedropena-producthackers/frontend-tech-tests/tree/main/prueba-c-nextjs-newsletter) |

> **Nota:** la Prueba C requiere el entorno **Cloud Sandbox / Devbox** de CodeSandbox (no el Browser Sandbox) porque los Server Actions necesitan Node real.

## Preguntas de profundización (cualquier prueba)

- ¿Por qué tomaste esta decisión en vez de la alternativa?
- ¿Qué pasaría si el usuario hace X? ¿Lo has contemplado?
- ¿Cómo lo llevarías a producción? ¿Qué cambiarías?
- ¿Qué parte mejorarías si tuvieras 10 minutos más?
